# R4.6 Native Library UI Deprecation Plan

## Status

R4.6 records the gradual deprecation of the Native ChatGPT Library UI in
favor of Desktop Studio. R4.6.0 (this slice) installs reversible flag-
based plumbing — **zero user-visible change by default**. Later slices
flip default values, add deprecation banners, and (in R4.7+) physically
remove the deprecated UI code.

Gate decision (R4.6.0): the deprecation MECHANISM is installed and
verifiable. The deprecation BEHAVIOR is not yet activated.

## Hard invariants — what R4.6 must NEVER break

These paths are explicitly NOT gated by any deprecation flag and remain
unconditional across all R4.6.x slices:

- **Capture path** — `H2O.LibraryActions.addToLibrary` /
  `H2O.LibraryActions.saveToFolder` / `H2O.LibraryActions.openLinkedChat`
  remain callable. The "Add to Library" / "Save to Folder" chat-row
  menu injections in 0F3a (`ENGINE_injectAddToLibrary` /
  `ENGINE_injectAddToFolder`) fire on every chat-row "..." menu open.
- **Save UX** — 0D3d Save Strip continues to appear after capture.
- **Tag extraction** — 0F5a remains the canonical owner of turn-level
  tag extraction. The MutationObserver and conversation-turn observation
  patterns stay live; the file is not modified by R4.6.
- **MV3 fallback APIs** — `renameCategory`/`deleteCategory`/
  `createCategory` (0F4a), `renameLabel`/`deleteLabel`/`createLabel`
  (0F6a), and `openFolderCreatePanel` (0F3a) remain unconditional.
  These are the fallback ladder for Chrome Studio's S0Z1g re-wiring
  (R4.5.1.a–R4.5.3).
- **Chrome → Desktop mirror** — R3 sync paths (auto-import.mv3.js,
  focus-import.tauri.js, folder-sync.tauri.js) are not touched.
- **Studio R4.5 surfaces** — S0F1m / S0F1n / S0Z1g re-wiring all
  continue to work; R4.5.5 matrix validator continues to pass.

If R4.6.x ever appears to threaten any of these, the change must be
deferred until the invariant is preserved.

## Flags

| Flag | Default (R4.6.0) | Controls |
|---|---|---|
| `library.nativeWorkspaceUi` | `true` (no change) | Items 1, 2, 3 |
| `library.nativeOrganizationUi` | `true` (no change) | Items 4, 5, 8, 10 |
| `library.nativeCaptureOnlyMode` | `false` | Operator-facing convenience (composed) |

All flags default to values that preserve **pre-R4.6 behavior**.
Operators can flip via DevTools:
```
H2O.flags.set('library.nativeOrganizationUi', false)
H2O.flags.set('library.nativeWorkspaceUi',    false)
```

`library.nativeCaptureOnlyMode = true` is conceptually equivalent to
setting both above to `false`. In R4.6.0 it is read-only/diagnostic;
the composed semantics ship in R4.6.1+.

## Phased retirement plan (as implemented)

- **R4.6.0** — Plumbing only. Per-module flag-reader helpers + diagnose
  entries + 0F1b code-level gate at `mountPage`. Validator established.
  Defaults `true` — zero user-visible change. **DONE — commit `4dcab8d`.**
- **R4.6.1** — Deprecation banner inside `mountPage` + body-attribute
  CSS gates per module + restore-flag button. **DONE — commit `4dcab8d`.**
- **R4.6.2** — Replace placeholder selectors with real ones
  (folder-row/-more, lbsc-root, .ho-project-row). **DONE — commit `4dcab8d`.**
- **R4.6.3** — Cascade-proof per-element gate. `body[data-h2o-r46-hide-*]`
  descendant rule was unreliable in production (host framework strips
  body attributes mid-render). Replaced with per-element marking +
  `el.style.setProperty('display','none','important')` (CSSOM inline
  `!important` — definitively wins over any cascade or inline style).
  **DONE — commit `fa70892`.**
- **R4.6.4** (THIS SLICE) — **Default flag flip.** Defaults change in
  `0F1k`'s `NATIVE_FLAG_DEFAULTS` table:
    `library.nativeWorkspaceUi:    false`
    `library.nativeOrganizationUi: false`
    `library.nativeCaptureOnlyMode: true`
  Operators who haven't explicitly set the flags now see the Native
  Library UI HIDDEN by default; capture/extraction/MV3 fallback APIs
  remain unchanged. Restore escape hatch (`H2O.flags.set(...,true)` +
  reload) still works because storage values win over defaults.
  **This is the irrevocable user-visible deprecation step.**
- **R4.6.5** — Release gate validator + doc closes R4.6 (next slice).
- **R4.7+** — Physical code removal of the deprecated UI mount paths
  into `src-runtime-base/retired/r4.6-native-library-ui/`. Soak-proven;
  safe to delete. Reduces bundle size. **NOT in R4.6.4.**

### Rollback (operator-level)

To restore the Native Library UI temporarily, paste in DevTools on
chatgpt.com:

```js
H2O.flags.set('library.nativeWorkspaceUi',    true);
H2O.flags.set('library.nativeOrganizationUi', true);
location.reload();
```

The R4.6.1 banner's "Restore Native Library UI (temporary)" button
does exactly this. The values get persisted to localStorage, so they
win over the R4.6.4 defaults on subsequent loads.

### Rollback (source-level — emergency)

If the R4.6.4 flip needs to be reverted globally, revert commit
`<R4.6.4 commit hash>` — the only change is the `NATIVE_FLAG_DEFAULTS`
table in 0F1k. No code paths were removed; reverting the table
restores defaults to caller-supplied fallback (true for workspace +
org), which is the pre-flip behavior.

## Archive folder plan — DEFERRED

When R4.7+ physically removes the deprecated UI code paths, the
removed sources should land in `src-runtime-base/retired/r4.6-native-
library-ui/` for archival (matching the pattern used by prior
deprecations). R4.6 does NOT create this folder yet. The folder is
introduced in R4.7's first removal slice.

## Validator

`tools/validation/native/validate-native-library-deprecation.mjs`
asserts:

- **A. Flag names referenced** — each gated Native module references
  the appropriate flag string literal (`library.nativeWorkspaceUi`
  or `library.nativeOrganizationUi`).
- **B. Flag-reader helpers present** — each gated module declares a
  helper that reads the flag with default `true` via
  `H2O.flags?.get(flag, true)`.
- **C. 0F1b code-level gate** — `mountPage` (the workspace page entry
  point) has a flag check at its top using the helper.
- **D. Capture path unconditional** — 0F1j's `addToLibrary` and
  `saveToFolder`, 0F3a's `ENGINE_injectAddToLibrary` and
  `ENGINE_injectAddToFolder`, are NOT preceded by any deprecation
  flag check.
- **E. Folder-create panel unconditional** — `openFolderCreatePanel`
  in 0F3a is NOT gated.
- **F. MV3 fallback APIs unconditional** — 0F4a's
  `renameCategory`/`deleteCategory`/`createCategory` and 0F6a's
  `renameLabel`/`deleteLabel`/`createLabel` are NOT gated.
- **G. 0F5a untouched** — file size unchanged from pre-R4.6 baseline;
  MutationObserver / conversation-turn observation patterns intact.
- **H. Diagnose surfaces** — each gated module's `diagnose()` reports
  the current flag state under a `flags` or `deprecation` sub-key.
- **I. R4.5.5 not regressed** — cross-check by re-running the
  organization-UI matrix validator at the end of R4.6 CI.

Expected count after R4.6.0: ~50 assertions in the native deprecation
validator; R4.5.5 stays 107/0; library-actions 135/0; import-bundle
277/0; import-graph clean.

## Hard constraints (R4.6.0)

- No file retirement, no archive folder creation.
- No default flag flip (all flags default to `true`/`false` as listed).
- No Native capture changes, no Native tag extraction changes.
- No schema changes.
- No Studio R4.5 file changes.
- No commits unless approved.
