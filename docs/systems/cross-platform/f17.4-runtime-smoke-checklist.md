# F17.4 Runtime Smoke Checklist

## Architecture Verdict

F17.4 is irreducibly manual and docs-only. It covers the release risks that Node validators cannot prove:

- Live Tauri runtime.
- Real SQLite path.
- WKWebView boot.
- Real Studio behavior.
- Extension-vs-Native-ChatGPT behavior if the extension ships.

The F17.4 output is a signed/manual operator attestation for F17.5 evidence. It should clear or explicitly cover:

- `studio-runtime-not-attested`
- `migration-rollback-live-db-not-attested`

## Runtime Smoke Sections

The operator should run and record results for:

- Desktop Studio boot.
- Library smoke.
- Folder smoke.
- Sync/runtime proof smoke.
- Release validators.
- Migration/backup attestation.
- Extension smoke if the extension ships.

## Desktop Studio Boot Checklist

- App launches to Studio.
- Redirector to `studio.html` works.
- DevTools console has no blocking errors.
- Tauri/Rust log has no blocking panic.
- App version is known or visible if available.
- Migrating DB launch completes without schema error.

## Library Checklist

- Library opens.
- Saved/imported chats are visible.
- Linked/saved reader opens.
- Imports are not broken.
- No unexpected blank states appear.
- Counts are plausible compared with the prior session.

## Folder Checklist

- Folder color works.
- Folder rename works.
- Folder delete works.
- Chat-folder bind works.
- Chat-folder unbind works.
- F7 fallback remains compatible.
- Guarded `folder_bindings` trigger does not break normal folder UX.
- Trigger guard toggle does not corrupt folder state.

## Manual DevTools Commands

Run these in Desktop Studio DevTools:

```js
await H2O.Desktop.Sync.runLibraryEndToEndSyncProof()
await H2O.Desktop.Sync.runLibrarySyncClosureProof()
await H2O.Desktop.Sync.runLibraryMultiPeerSoakProof()
await H2O.Desktop.Sync.runLibraryPerformanceStressProof()
await H2O.Desktop.Sync.runLibraryFolderBindingAbsorptionProof()
await H2O.Desktop.Sync.runLibraryFolderBindingTriggerProof?.()
```

Expected results:

- Each core proof returns `ok: true`.
- `sideEffectSummary.realBusinessTableWritten === false` when that field is present.
- `runLibraryFolderBindingTriggerProof?.()` may be `undefined`; a standalone command is not currently guaranteed.
- Guarded trigger coverage lives in the closure proof and folder absorption proof.

## Terminal Validator Commands

Run:

```sh
npm run gate:library
npm run audit:secrets
npm run validate:build
npm run validate:migration
```

Expected results:

- No blockers.
- Warnings are reviewed and either cleared or accepted in the operator attestation.

## Backup / Rollback Attestation

Before migration-sensitive launch or update, the operator must attest:

- Manual `.bak` of `studio-v1.db` was taken.
- `studio-v1.db-wal` and `studio-v1.db-shm` were copied if present.
- Backup set is stored in a known timestamped location.
- Restore path is known: close app, close DB pool, replace DB files, relaunch.
- Trigger guard disable lever is known: disable `f16_folder_bindings_trigger_guard.enabled`.
- F15 delegation disable lever is known: `setF15FolderBindingDelegationEnabled(false)`.
- Operator acknowledges runtime auto-backup is deferred.

## Extension Smoke If Extension Ships

- Extension loads cleanly.
- Correct production variant is installed.
- Native ChatGPT still works.
- OAuth/prod identity works.
- Extension console has no blocking errors.
- Active build does not use test, dev, or staging endpoint.
- Cross-check extension artifacts with `npm run audit:secrets -- --artifacts`.

If the extension is not shipped in the release, record `extension not shipped`.

## Blocker / Warning Taxonomy

Smoke blockers:

- App boot fails.
- Console/Rust blocking error.
- Library missing or blank.
- Folder color/rename/delete/bind/unbind regression.
- Guarded trigger breaks normal folder UX.
- Any DevTools proof returns `ok: false`.
- Any `realBusinessTableWritten: true`.
- Any release validator is blocked.
- Manual backup not completed before migrating launch.
- Extension breaks Native ChatGPT.
- OAuth fails.
- Test endpoint appears in active extension build.

Smoke warnings:

- Heavy soak/stress not run.
- `--cargo` or `--tauri-build` not run.
- Validators return warnings only.
- `migration-rollback-plugin-db-path-runtime-decision-required`.
- `runLibraryFolderBindingTriggerProof` is undefined.
- Minor documented non-blocking console noise.

## Risks And Mitigations

| Risk | Mitigation |
|---|---|
| Node validators are green but live runtime fails | Require Desktop Studio DevTools proof commands and boot smoke |
| Manual backup is skipped | Require explicit backup attestation for F17.5 evidence |
| Extension breaks Native ChatGPT | Run extension smoke only for shipped extension variants |
| Folder operations regress | Exercise color, rename, delete, bind, unbind, fallback, and guarded trigger UX |
| Proof command unavailable in live runtime | Treat required core proof absence as blocker; document optional trigger proof absence as warning |
| Operator evidence is incomplete | Require timestamp, commit, platform, outputs, warnings, failures, verdict, and signature |
| WKWebView boot race causes transient failure | Clean relaunch and rerun proof once; persistent failure remains blocker |
