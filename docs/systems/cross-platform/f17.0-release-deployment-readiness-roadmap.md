# F17.0 Release / Production Deployment Readiness Roadmap

## Architecture Verdict

F17 is a release and production deployment readiness phase, not a new sync-feature phase.

F16 is the architecture freeze line for Library Sync hardening. F17 wraps the hardened F15/F16 sync architecture in a deployable release process by reusing existing release, build, archive, validation, and release-gate tooling.

F17 reuses:

- `tools/release/*`
- `tools/validation/identity/run-identity-release-gate.mjs`
- `tools/validation/sync/run-f16-production-release-gate.mjs`
- `npm run gate:library`

F17 fills three concrete release-readiness gaps:

- No dedicated credential / secret scanner.
- No DB-backup-before-migration mechanism.
- No rollback drill.

## Release Readiness Scope

F17 covers these release surfaces:

- Desktop Studio / Tauri.
- Chrome MV3 extension.
- Shared runtime base.
- Frozen `sync/library` modules.
- SQLite migrations `v1` through `v13`.
- Release validators.
- Build and release scripts.
- Operator docs.

Desktop Studio is local-first SQLite and has no Supabase or Stripe cloud secret surface. Chrome MV3 identity/OAuth is the real secret-sensitive surface.

F17 does not add sync features. F17 should not introduce schema changes except safety or release-readiness work. Retirement of `f16.folder-legacy-fallback` is deferred to a later phase, likely F18.

## Environment / Secrets Checklist

F17.1 should verify:

- No Supabase or Stripe keys in Tauri, runtime, or surface trees.
- OAuth/identity secrets are limited to the Chrome extension build surface and release configuration.
- No dev/test tokens in shipped artifacts.
- No localhost or staging endpoints in production build artifacts.
- `.env` files are ignored and not shipped.
- Secret rotation is planned before production.
- CI/CD secret injection does not log secret values.

## Migration / Rollback Checklist

F17.3 should verify:

- SQLite migrations `v1` through `v13` apply sequentially.
- `v13` guarded `folder_bindings` trigger infrastructure remains default-safe.
- `f16_folder_bindings_trigger_guard.enabled` can be inspected and controlled.
- Writer identity sentinel remains available.
- Allowed active trigger identities remain:
  - `f15.execute-settlement-writer`
  - `f16.folder-legacy-fallback`
- Migration idempotency is preserved.
- Old user DB compatibility is covered.
- Backup `.bak` is created before migration.
- Rollback levers are documented and drilled:
  - Disable trigger guard.
  - Disable F15 delegation.
  - Restore DB backup.
  - Reinstall prior artifact.

## Manual Runtime Smoke Checklist

F17.4 should require:

- Desktop Studio boots cleanly.
- No console errors.
- Library opens.
- Imported/saved chats are visible.
- Folder color, rename, and delete behavior remains intact.
- Chat-category and chat-folder binding works.
- Cache reflects binding truth.
- DevTools proofs return `ok: true`.
- `npm run gate:library` passes.
- If the Chrome extension is shipped, Native ChatGPT behavior is not broken and OAuth works against production configuration.

## Risks And Mitigations

| Risk | Mitigation |
|---|---|
| Shipping with test keys | Dedicated F17.1 secret/config audit |
| Migration breaks existing user DB | Backup-before-migration and old-DB compatibility proof |
| Tauri build mismatch | F17.2 build/package validation |
| Chrome/Desktop divergence | Cross-surface package and runtime smoke checks |
| Release gate passes but live runtime fails | Manual Studio DevTools runtime smoke checklist |
| Unrelated WIP included in release | Release gate git-state checks and final audit |
| Rollback path untested | F17.3 rollback drill and F17.5 evidence |
| No auto-updater / stale installs | Document release distribution and reinstall expectations |
| Evidence bundle drift | Generated evidence bundle with command outputs and commit identifiers |
