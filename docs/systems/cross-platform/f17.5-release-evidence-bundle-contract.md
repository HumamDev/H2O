# F17.5 Release Evidence Bundle Contract

## Architecture Verdict

F17.5 is a collector/aggregator phase. It should:

- Invoke existing `--json` validators.
- Merge their structured outputs with the F17.4 manual runtime attestation.
- Add git, version, and platform metadata.
- Emit one dated, redacted evidence bundle.
- Re-run no validation logic.
- Become the single artifact audited by F17.6.

The collector is not a new validator. Pass/fail authority stays with the existing validators and the signed operator attestation.

## Machine-Readable Evidence Sources

Required machine-readable sources:

| Command | Schema |
|---|---|
| `npm run gate:library -- --json` | `h2o.desktop.sync.library-production-release-gate.v1` |
| `npm run audit:secrets -- --json` | `h2o.release.secret-config-audit.v1` |
| `npm run validate:build -- --json` | `h2o.release.build-package-validation.v1` |
| `npm run validate:migration -- --json` | `h2o.release.migration-rollback-validation.v1` |

The collector should execute these commands and persist their JSON outputs as evidence inputs. It must not infer success by scraping human-readable logs.

## Manual Evidence Sources

Required manual sources:

- F17.4 runtime smoke attestation.
- DevTools proof outputs.
- Backup/manual rollback attestation.
- Extension smoke result if the extension ships.
- Optional heavy/cargo/tauri-build/studio-attested results if run.

Missing or incomplete F17.4 attestation means the bundle is `RELEASE EVIDENCE INCOMPLETE`, never silently ready.

## Evidence Content Table

| Source | Auto/manual | Required/optional | Inclusion format | Redaction behavior |
|---|---|---|---|---|
| `gate:library -- --json` | Auto | Required | Full JSON result plus summarized rollup | Keep validator output; scan bundle for forbidden needles |
| `audit:secrets -- --json` | Auto | Required | Reduced counts, finding codes, severity, allowlist count | Never include matched secret values |
| `validate:build -- --json` | Auto | Required | Full JSON result plus version/build summary | Keep artifact paths only when non-sensitive |
| `validate:migration -- --json` | Auto | Required | Full JSON result plus migration/rollback summary | Include manifest metadata only; no DB or backup contents |
| F17.4 attestation | Manual | Required | Operator-provided JSON or markdown parsed to structured fields | Redact logs/screenshots before inclusion |
| DevTools proof outputs | Manual | Required | Reduced proof summaries | Keep `ok`, `version`, case counts, side-effect flags |
| Backup attestation | Manual | Required | Backup path string, manifest hash/metadata, operator signature | Never include raw `.bak`, `.db`, `-wal`, or `-shm` |
| Extension smoke | Manual | Required if extension ships | Result, variant, OAuth/native smoke status | Never include private config or tokens |
| Optional heavy/cargo/tauri-build | Mixed | Optional | Command result summaries | Include pass/warn/fail state and command metadata |

## Output Schema

Use:

```text
h2o.release.evidence-bundle.v1
```

Required shape:

```js
{
  schema,
  version,
  generatedAtIso,
  release: {
    headCommit,
    appVersion,
    branch,
    treeClean,
    stagedApproved
  },
  platform: {
    os,
    arch,
    node,
    operatorId
  },
  validators: {
    gateLibrary,
    secretAudit,
    buildPackage,
    migrationRollback
  },
  runtimeSmoke,
  backup,
  extension,
  rollup: {
    verdict,
    blockers,
    warnings,
    incompleteFields
  },
  observedAtIso
}
```

Rollup verdicts:

- `RELEASE EVIDENCE READY`
- `RELEASE EVIDENCE CONDITIONAL`
- `RELEASE EVIDENCE INCOMPLETE`
- `RELEASE EVIDENCE BLOCKED`

## Bundle Output Location

Use an out-of-tree dated sibling directory, following the `tools/archive/archive-snapshot.mjs` precedent.

Recommended location:

```text
../release-evidence/<YYYY-MM-DD>/<headShortSha>/
```

Bundle contents:

- `evidence-bundle.json`
- `evidence-report.md`
- Copies of the four validator JSONs.
- Optional zip: `evidence-<sha>-<date>.zip`

The collector must refuse to write inside the source tree.

## Redaction / Privacy Rules

Never include:

- Raw `.bak`.
- Raw `.db`.
- `-wal`.
- `-shm`.
- Extension private config.
- `config/local/*`.
- `.env`.
- Raw tokens/keys.
- Raw DB files.
- Raw secret matched values.

Secret audit sub-result should be reduced to:

- Counts.
- Finding codes.
- Severity.
- No matched secret values.

Backup attestation may carry:

- Backup path string only.
- Manifest hash/metadata only.
- Never backup contents.

DevTools proof outputs should keep:

- `ok`.
- `version`.
- Case counts.
- `realBusinessTableWritten`.
- Side-effect flags.

The collector should run a full-bundle redaction/needle scan using F17.1-style forbidden needles and block if raw leaks appear.

## Required Manual Attestation Behavior

- `--attestation` is required for a complete bundle.
- Missing attestation produces `RELEASE EVIDENCE INCOMPLETE`.
- Stale attestation HEAD mismatch is a warning or blocker depending on severity.
- Operator identity/signature is required.
- Runtime smoke result is required.
- Backup attestation is required.
- Extension smoke is required only if the extension ships.

## Risks And Mitigations

| Risk | Mitigation |
|---|---|
| Secret value leaks into bundle | Reduce secret audit result and run full-bundle forbidden-needle scan |
| Raw DB or backup files are included | Collector refuses `.db`, `.bak`, `-wal`, and `-shm` inputs |
| Bundle is accidentally committed to repo | Collector writes only to `../release-evidence/...` and refuses source-tree output |
| Missing or forged attestation | Require operator identity/signature and HEAD match checks |
| Validator schema drift | Require exact schema strings for the four JSON validator inputs |
| Stale evidence from a different commit | Compare bundle HEAD, attestation HEAD, and validator HEAD metadata when present |
| Optional checks not run but reported ready | Preserve optional states and include missing optional checks as warnings |
| Collector mutates repo or DB | Collector is read-only except out-of-tree evidence output |
