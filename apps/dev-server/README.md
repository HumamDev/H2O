# apps/dev-server — Python HTTP server (legacy chatgpt+chrome dev pipeline)

A tiny Python HTTP server that serves the alias farm + extension proxy pack
consumed by the legacy chatgpt+chrome unpacked development extensions when
they run on `https://chatgpt.com/`. Listens on `http://127.0.0.1:5500` by
default.

The dev-server is part of the **legacy chatgpt+chrome runtime path**
established before the multi-host migration. The new-host stubs
(`claude/chrome`, `chatgpt/firefox`, `claude/firefox`, `gemini/chrome`,
`gemini/firefox`) do NOT use this dev-server — they ship their content
script directly inside the unpacked extension and don't need a local proxy.

## Source vs generated — what's safe to edit, what's not

(Added in Phase 8H-0 for clarity.)

### ✅ Source — edit these freely

| Path | What |
|---|---|
| `serve.py` | The HTTP server. Pure Python; CORS-allow-all + no-cache headers; serves the current working directory. The only tracked file in this folder. |

### ❌ Generated — never edit, never commit

| Path | Generated by | Purpose |
|---|---|---|
| `alias/` | `tools/loader/make-aliases.mjs` | 151+ symlinks from numeric-ID aliases to the actual `src-runtime-base/*.user.js` files (folder renamed from `scripts/` in Phase 8K-5). Regenerated on every dev rebuild. Symlinks are relative (`../../../src-runtime-base/…`) so they survive folder renames. |
| `dev_output/` | `tools/loader/make-ext-proxy-pack.mjs` | The proxy-pack text file (`dev_output/proxy/_paste-pack.ext.txt`) that legacy chatgpt+chrome extensions fetch at boot to discover which scripts to load and in what order. |

Both are covered by gitignore rules (`apps/dev-server/alias/**`,
`apps/dev-server/dev_output/**`).

## How it's used

The legacy chatgpt+chrome dev extensions fetch this URL at boot:

```
http://127.0.0.1:5500/dev_output/proxy/_paste-pack.ext.txt
```

That file lists the alias-farm script URLs in dev-order. The extension's
loader then fetches each script via:

```
http://127.0.0.1:5500/alias/{aliasId}.js?v={timestamp}
```

The dev-server simply serves the current directory as static files; the
proxy pack URL contract + the alias farm structure together implement the
runtime loading.

## Start the server

```sh
cd apps/dev-server
python3 serve.py 5500
```

(Or set `H2O_SERVER_DIR=/path/to/this/folder` and run from anywhere.)

## Operator notes

- The HTTP serving endpoint URL `http://127.0.0.1:5500/...` is a contract
  baked into the legacy chatgpt+chrome manifests (`H2O_EXT_PROXY_PACK_URL`).
  Changing the port requires a coordinated rebuild of all dev extension
  variants.
- This server is **chatgpt+chrome legacy only** — new host/browser stubs
  load their bundled content script directly via Chrome MV3 content_scripts
  or Firefox WebExtension content_scripts, with no runtime proxy.
- Phase 5B (2026-05-17) absorbed this from a previously-sibling
  `cockpit-pro/h2o-dev-server/` directory; the URL contract did not change.
