# apps/studio/mobile/ios — iOS managed native project

This is the **iOS half of the Studio Mobile app**, managed jointly by
Expo (autolinking + `expo run:ios`), CocoaPods (development pods), and
Xcode (build settings, signing). It is a **hybrid** of source and
generated content — read this README before editing anything inside.

## Source vs generated — what's safe to edit, what's not

(Established in Phase 8H-0; expanded in Phase 8H-2 with this per-folder
boundary doc, matching the convention used for `apps/dev-server/` and
the extension folders.)

### ✅ Source — edit these freely

| Path | What |
|---|---|
| `Podfile` | CocoaPods spec for the workspace; Expo + RN scripts are required from here. The trailing `post_install` block also patches `ExpoModulesJSI.debug/release.xcconfig` to set `CLANG_CXX_LANGUAGE_STANDARD = c++20` (Phase 8H-0 workaround for an upstream omission in expo-modules-core 55.0.25). |
| `Podfile.properties.json` | Hermes/JS engine + dev-client toggles. |
| `H2OStudio.xcodeproj/project.pbxproj` | Xcode project file (build settings, targets, file refs). Edited via Xcode UI or scripted patches. |
| `H2OStudio.xcodeproj/project.xcworkspace/contents.xcworkspacedata` | Workspace declaration. |
| `H2OStudio.xcodeproj/project.xcworkspace/xcshareddata/IDEWorkspaceChecks.plist` | Shared IDE settings. |
| `H2OStudio.xcodeproj/xcshareddata/xcschemes/H2OStudio.xcscheme` | Shared build scheme. |
| `H2OStudio.xcworkspace/contents.xcworkspacedata` | CocoaPods-generated workspace reference. |
| `H2OStudio/AppDelegate.swift` | App entry point (RN / Expo bootstrap). |
| `H2OStudio/H2OStudio-Bridging-Header.h` | Swift ↔ Obj-C bridging header. |
| `H2OStudio/Info.plist` | App metadata, URL schemes, capabilities, permission strings. |
| `H2OStudio/H2OStudio.entitlements` | App entitlements (App Groups, share extension, push, etc.). |
| `H2OStudio/PrivacyInfo.xcprivacy` | Apple Privacy Manifest. |
| `H2OStudio/Images.xcassets/` | App icon + splash colour set. |
| `H2OStudio/SplashScreen.storyboard` | Launch storyboard. |
| `H2OStudio/expo.icon/` | Expo icon source (`icon.json` + `Assets/*`). |
| `H2OStudio/Supporting/Expo.plist` | Expo runtime plist. |
| `H2OShareExtension/Info.plist` | Share-sheet extension metadata. |
| `H2OShareExtension/H2OShareExtension.entitlements` | Share-sheet entitlements (App Group binding). |
| `H2OShareExtension/ShareViewController.swift` | Share-sheet entry point. |
| `.xcode.env` | Project-level Xcode environment (committed). |

`Podfile.lock` is also tracked — it pins exact pod versions to keep
builds deterministic across operators. Regenerated by `pod install`; the
diff should be reviewed but not hand-edited.

### ❌ Generated — never edit, never commit

These are all gitignored (root `.gitignore` `apps/**/...` rules and
`apps/studio/mobile/ios/.gitignore`).

| Path | Generated by | Notes |
|---|---|---|
| `Pods/` | `pod install` | Includes generated `.xcconfig` files; do not hand-edit (changes are overwritten on next `pod install`). Patch via `Podfile` `post_install` hooks instead. |
| `build/` | Xcode | Intermediate build outputs. |
| `**/xcuserdata/`, `**/*.xcuserstate` | Xcode | Per-user IDE state (window positions, breakpoints). |
| `.xcode.env.local` | Operator / Xcode | Per-machine env (e.g. `NODE_BINARY=/usr/local/bin/node`). |

### ⚠ Special case — `H2OStudio.xcodeproj/project.pbxproj`

Although tracked as source, this file is also touched by:

- `pod install` (links development pods, regenerates the `Pods` group)
- Xcode UI (adding files, changing build settings)
- Expo prebuild (only when re-run; the project is now hand-managed after
  the initial prebuild)

When reviewing diffs to this file, scrutinize whether the change is
intentional (a new file added, a setting changed) versus accidental
churn from `pod install` re-sorting groups.

### Quick verification

```sh
# Source files (tracked):
git ls-files apps/studio/mobile/ios/ | head

# Generated artifacts (ignored):
git status --ignored apps/studio/mobile/ios/ | head
```
