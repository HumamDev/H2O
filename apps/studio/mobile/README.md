# H2O Studio — Mobile App

iPhone-first React Native / Expo app for browsing, searching, and reading your saved ChatGPT conversations.

## Stack

- Expo SDK 55 + Expo Router (file-based routing)
- React Native 0.83
- TypeScript (strict)
- SQLite via `expo-sqlite` (planned)

## Getting started

```bash
cd apps/studio/mobile
npm install
npm run ios        # open in iOS Simulator
npm run android    # open in Android emulator
npm run web        # run in browser
```

## Source vs generated — what's safe to edit, what's not

(Added in Phase 8H-0 for clarity. Matches the source/generated boundary
established for the extension architecture.)

### ✅ Source — edit these freely

| Path | What |
|---|---|
| `src/` | TypeScript source: `app/` (Expo Router routes), `components/`, `features/`, `db/`, `repositories/`, `parser/`, `renderer/`, `importers/`, `exporters/`, `search/`, `sync/`, `theme/` |
| `assets/` | Icons, fonts, images bundled into the app |
| `__tests__/` | Test scaffolding |
| `app.json` | Expo app config (slug, scheme, bundle ID, plugins) |
| `babel.config.js` | Babel/Metro transformer config |
| `metro.config.js` | Metro bundler workspace-root config (Phase 6C-3 depth-aware) |
| `tsconfig.json` | TypeScript strict mode + path aliases |
| `eslint.config.js`, `eas.json` | Lint + EAS Build config |
| `package.json` | npm scripts + dependencies |

### ⚠ Managed native projects — partly source, partly generated

These directories are a hybrid. **Edit with care.**

| Path | Status | Reason |
|---|---|---|
| `ios/` | Managed by Expo + CocoaPods + Xcode | The xcodeproj structure is checked in, but `ios/Pods/`, `ios/build/`, `ios/**/xcuserdata/`, `ios/**/*.xcuserstate` are regenerated by `pod install` / Xcode and should NOT be edited or committed. Hand-edits to `Info.plist`, entitlements, or the Podfile are valid source changes; everything else under `ios/` is best treated as generated. |
| `android/` (future) | Same pattern when added | `android/build/`, `android/app/build/`, `android/.gradle/` will be generated by Gradle. |

### ❌ Generated / cache — never edit, never commit

These are all gitignored (see `.gitignore` rules in `apps/**/...` patterns):

| Path | Generated by |
|---|---|
| `.expo/` | Expo CLI (config caching, dev artifacts) |
| `ios/Pods/` | `pod install` |
| `ios/build/` | Xcode |
| `ios/**/xcuserdata/`, `ios/**/*.xcuserstate` | Xcode (per-user IDE state) |
| `ios/.xcode.env.local` | Xcode (per-machine env) |
| `android/build/`, `android/app/build/`, `android/.gradle/` (future) | Gradle |
| `node_modules/` | `npm install` |
| `dist/`, `web-build/` (if Expo web is enabled) | Expo bundler |
| `expo-env.d.ts` | Expo TypeScript shim (regenerated; harmless to commit but not edited) |
| `*.tsbuildinfo` | TypeScript incremental build cache |
| `.env*.local` | Operator-only secrets |

### Quick verification

```sh
# All tracked files under apps/studio/mobile/ (should be source only):
git ls-files apps/studio/mobile/ | head

# All gitignored paths (caches, native builds, secrets):
git status --ignored apps/studio/mobile/ | head
```

If `git status` ever shows a file under `.expo/`, `ios/Pods/`, `ios/build/`, or
similar — that's a sign a `.gitignore` rule needs to be added. The
`apps/**/...` defense-in-depth rules already cover the common cases.
